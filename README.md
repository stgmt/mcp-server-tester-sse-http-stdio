## üéØ Why This Tool?

### The Problem
- üî¥ **Self-written test scripts** - Every team reinvents the wheel with custom scripts
- üî¥ **No unified architecture** - Fragmented testing approaches across different transports
- üî¥ **Protocol complexity** - SSE, HTTP, and STDIO each require different handling
- üî¥ **Maintenance nightmare** - Custom scripts break with protocol updates
- üî¥ **Poor readability** - Complex test logic buried in imperative code

### Our Solution
- ‚úÖ **Professional architecture** - Well-designed, extensible testing framework
- ‚úÖ **All MCP protocols** - Complete support for every existing MCP transport
- ‚úÖ **Declarative YAML tests** - Describe test scenarios following BDD best practices
- ‚úÖ **Scenario-based testing** - Write tests as user stories with Given-When-Then structure
- ‚úÖ **Real SSE support** - Native EventSource protocol, not just HTTP POST
- ‚úÖ **TypeScript safety** - Catch errors at compile time
- ‚úÖ **One-line execution** - `npx tsx src/cli.ts tools test.yaml --server-config server.json`
- ‚úÖ **CI/CD ready** - JUnit XML output for any pipeline

## üöÄ Quick Start

### Node.js/TypeScript

#### 1. Create test file (test.yaml)
```yaml
tools:
  tests:
    - name: Test server status
      tool: get_status
      params: {}
      expect:
        success: true
        result:
          contains: "status"
```

#### 2. Create server config (server.json)  
```json
{
  "mcpServers": {
    "my-server": {
      "transport": "sse",
      "url": "http://localhost:8001/sse"
    }
  }
}
```

#### 3. Run tests
```bash
# Install globally
npm install -g mcp-server-tester-sse-http-stdio

# Run tests (‚úÖ ALWAYS specify --server-name!)
npx mcp-server-tester-sse-http-stdio tools test.yaml --server-config server.json --server-name my-server
```

‚úÖ **Server must be running** on specified URL for tests to pass!

#### Test Any Transport in Seconds

**HTTP:**
```bash
npx mcp-server-tester-sse-http-stdio tools examples/crawl4ai-http-tests.yaml \
  --server-config examples/crawl4ai-http-config.json --server-name crawl4ai
```

**SSE:**
```bash
npx mcp-server-tester-sse-http-stdio tools examples/graphiti-sse-tests.yaml \
  --server-config examples/graphiti-sse-config.json --server-name graphiti
```

**STDIO:**
```bash
npx mcp-server-tester-sse-http-stdio tools examples/crawl4ai-stdio-tests.yaml \
  --server-config examples/crawl4ai-stdio-config.json --server-name crawl4ai
```

## üö® Common Pitfalls

### Multiple servers in config
‚ùå **Wrong:**
```bash
npx mcp-server-tester-sse-http-stdio tools test.yaml --server-config server.json
# Error: Multiple servers found, please specify server name
```

‚úÖ **Correct:**
```bash
npx mcp-server-tester-sse-http-stdio tools test.yaml --server-config server.json --server-name graphiti
```

### Server config format  
‚ùå **Wrong** (copying from MCP client configs):
```json
{
  "mcpServers": {
    "graphiti": {
      "transport": "sse",
      "url": "http://localhost:8001/sse",
      "description": "My server",  // ‚ùå Not supported
      "timeout": 30000             // ‚ùå Not supported
    }
  }
}
```

‚úÖ **Correct** (minimal format):
```json
{
  "mcpServers": {
    "graphiti": {
      "transport": "sse",
      "url": "http://localhost:8001/sse"
    }
  }
}
```

### Test result expectations
‚ùå **Wrong:**
```yaml
expect:
  success: true
  result:
    matches: true  # ‚ùå Not supported
```

‚úÖ **Correct:**
```yaml
expect:
  success: true
  result:
    contains: "expected text"  # ‚úÖ or use 'equals'
```

### Server not running
‚ùå **Error:**
```
Error: Failed to connect: ECONNREFUSED 127.0.0.1:8001
```

‚úÖ **Solution:**
1. Start your MCP server first
2. Verify server is accessible: `curl http://localhost:8001/health`
3. Check server URL in config matches actual server

```json

      "transport": "http|sse|stdio",
      "url": "http://localhost:3000",     // for HTTP/SSE

      "args": ["server.py", "--stdio"]     // for STDIO
    }
  }
}
```

## ‚öôÔ∏è Configuration

### Transport Types

| Transport | Use Case | Configuration |
|-----------|----------|---------------|
| **HTTP** | REST APIs | `"transport": "http", "url": "..."` |
| **SSE** | Real-time streaming | `"transport": "sse", "url": "..."` |
| **STDIO** | Local processes | `"transport": "stdio", "command": "..."` |

### Command Line Options

```bash
Options:
  --server-config <file>   MCP server configuration (required)
  --server-name <name>     Specific server from config
  --timeout <ms>           Test timeout (default: 10000)
  --debug                  Enable debug output
  --junit-xml [filename]   Generate JUnit XML report
  -t, --transport <type>   Override transport type
  -u, --url <url>          Override server URL
```
